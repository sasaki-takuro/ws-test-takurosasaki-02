## 本フォルダの内容

本フォルダは、AI活用ワークショップの演習ユースケース#1「案件概要資料の要点抽出と標準化」で使用する教材一式です。参加者向けの手順書、演習に使うダミーデータ4件、および参加者答え合わせ用の模範統合版MD 1件が含まれています。

> ⚠️ **【研修用模擬資料】本資料は演習目的で作成された架空データです。登場する企業・個人・数値・地域・組織名は全て架空であり、実在の企業・人物・案件とは一切関係ありません。**

## 演習の概要

- **所要時間**: 約45分
- **難易度**: 中上級
- **使用ツール**: Claude Code Web（Sonnet 4.6）＋ GitHub Web
- **業界バリエーション**: 小売（RT・都市型複合チェーン）
- **到達率**: 約65%（Agent◎）

### 業務の課題（要約）
仲介会社から届く案件概要資料はフォーマットや粒度がまちまちで、重要情報を手作業で拾っており1件2〜3時間要している。抽出品質も担当者の経験差でばらつく。

### ありたい姿（要約）
AIに投入するだけで企業情報・財務概要・拠点構成・人員体制・リスク・機会の各カテゴリに自動分類され、標準化出力される。1件2〜3時間 → 30分以内に短縮。

## 参加者の進め方

1. **準備（事前30〜40分）**: `preparation_section.docx` でGitHub/Claudeアカウント作成・Pro契約・GitHub連携・データ学習オプトアウトを完了
2. **Step1（約10分）**: Claude Code Webに過去3件を読み込ませ、`CLAUDE.md`に7カテゴリの抽出テンプレートを定義
3. **Step2（約20分）**: 新規1件（ベテルギウス小売）を対象に、`CLAUDE.md`に従って7セクションの結果を順次生成（1セクション1ターン厳守）
4. **Step3（約10分）**: 7ファイルをつなぎあわせて1ファイルの統合版MDを作成（8段階逐次追記方式）
5. **振り返り（約10分）**: 手順書§8のチェックポイントと模範統合版MDで到達度確認
