## 本フォルダの内容

本フォルダは、AI活用ワークショップの演習ユースケース#2「投資候補ロングリスト作成・スコアリング」で使用する教材一式です。参加者向けの手順書、演習に使うダミーデータ4件、および参加者答え合わせ用の模範統合版MD 1件が含まれています。

## 演習の概要

- **所要時間**: 約45分
- **難易度**: 中上級
- **使用ツール**: Claude Code Web（Sonnet 4.6）＋ GitHub Web
- **業界バリエーション**: 小売（RT・リテイル投資先候補）
- **到達率**: 約70%（Agent◎）
- **データ型**: 型B（表形式→スコアリング）

### 業務の課題（要約）
投資候補のロングリスト作成において、地域・売上規模・従業員数・業種等の複数軸の分類を手作業＋Excelで行っており、担当者依存。過去の投資/見送り判定ロジック（50〜100件規模）が文書化されておらず、再利用できない。

### ありたい姿（要約）
過去の投資判定データから抽出されたスコアリングロジック（重み付け・閾値・合否基準）が再利用可能なモデルとして確立。新規候補を入力すると多軸分類＋スコアリングが自動実行され、ランキングが即座に出力される体制。

## 参加者の進め方

1. **準備（事前30〜40分）**: `preparation_section.docx` でGitHub/Claudeアカウント作成・Pro契約・GitHub連携・データ学習オプトアウトを完了
2. **Step1（約10分）**: Claude Code Webに過去3件を読み込ませ、`CLAUDE.md`に5観点スコアリングロジック（地域・売上規模・利益率・業種適合・総合判定の重み付け・閾値・業態例外ルール）を定義
3. **Step2（約20分）**: 新規候補5件（カストル小売プロジェクト）を対象に、`CLAUDE.md`に従って7セクションの結果を順次生成（1セクション1ターン厳守）
4. **Step3（約10分）**: 7ファイルをつなぎあわせて1ファイルの統合版MDを作成（8段階逐次追記方式）
5. **振り返り（約10分）**: 手順書§8のチェックポイントと模範統合版MDで到達度確認

## 到達率70%の内訳

| 項目 | 機械判定可否 |
|---|---|
| 主要4軸（地域・規模・利益率・業種）のスコアリング | ✅ 高精度で機械化 |
| 業態例外ルール（アパレル・百貨店・小粒高収益） | ✅ CLAUDE.mdで明文化可能 |
| 判定3段階（投資/要検討/見送り） | ✅ 閾値ベースで機械判定可能 |
| 定性面（成長性・経営リスク・新業態持続性） | ⚠️ 残り30%の限界・人の判断で補完 |
| 新立地・新業態（過去判例なし） | ⚠️ スコアリング適用外・個別審議枠 |

> 💡 **実務規模での注意**: 本演習は5件規模ですが、実務ではClaude Codeデスクトップ版（ローカル実行）を使えば50〜100件も同じロジックで処理可能です。本教材の7セクション出力構成は件数が増えても変わりません。
