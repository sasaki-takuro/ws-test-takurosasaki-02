## 演習の概要

- **所要時間**: 約45分
- **難易度**: 中上級
- **使用ツール**: Claude Code Web（Sonnet 4.6）＋ GitHub Web
- **業界バリエーション**: 小売（RT・小売事業DCF）
- **到達率**: 約65%（Agent◎・軽量版）
- **データ型**: 型B派生（表形式→モデル構築）

### 業務の課題（要約）
企業価値算定のDCFモデル雛形が整備されておらず、案件ごとにゼロベースで作成。1モデルの構築に8〜16時間。PL/BS/CF・WACC・感度分析の標準化が属人的で、再利用が進まない。

### ありたい姿（要約）
過去の事業計画から抽出されたモデル構築ロジックに基づき、パラメータ入力だけで5年DCFモデルが自動生成。WACC算出と感度分析テーブルがワンステップで出力。構築時間8〜16時間→2時間以内に短縮。

## ⚠️ 本演習の重要な前提（到達率65%・5年軽量版）

本演習は**5年明示期間+ターミナル**の軽量版であり、以下は**対応不可・手動補正前提**です:

| 対応可（機械判定65%） | 対応不可（人の判断で補完） |
|---|---|
| 5年明示期間のPL構造 | 10年以上の超長期計画 |
| FCF算出（NOPAT起点・運転資本反映） | 季節変動・回転日数の年次変動を踏まえた精緻運転資本計算 |
| WACC算出（標準CAPM＋業界補正） | 多通貨・多国籍企業のWACC（カントリーリスクプレミアム含む） |
| ターミナルバリュー計算（Gordon） | 税効果会計・繰延税金・のれん償却・減損テスト |
| 感度分析2軸×5×5マトリクス + 3シナリオ | 3パラメータ×多次元感度分析（モンテカルロ等） |

> 💡 **実務規模での注意**: 本演習は5年軽量版ですが、実務ではClaude Codeデスクトップ版（ローカル実行）を使えば10年以上の長期モデル、3軸以上の感度分析、税効果の精緻計算も処理可能です。Web版は出力制限があるため、研修では5年軽量版で流れを体験することに集中します。

## 参加者の進め方

1. **準備（事前30〜40分）**: `preparation_section.docx` でGitHub/Claudeアカウント作成・Pro契約・GitHub連携・データ学習オプトアウトを完了
2. **Step1（約10分）**: Claude Code Webに過去3件を読み込ませ、`CLAUDE.md`にバリュエーションロジック（前提条件・PL/CF構造・WACC算定式・感度分析軸・モデル種別ごとの対応ルール）を定義
3. **Step2（約20分）**: 新規案件パラメータシート（アルビレオ小売）を対象に、`CLAUDE.md`に従って7セクションの結果を順次生成（1セクション1ターン厳守・**Pythonスクリプトは作成せずMarkdownの表と文章で計算過程を書き切る**）
4. **Step3（約10分）**: 7ファイルをつなぎあわせて1ファイルの統合版MDを作成（8段階逐次追記方式）
5. **振り返り（約10分）**: 手順書§8のチェックポイントと模範統合版MDで到達度確認

## 本UC#3 の最重要ルール: Pythonスクリプト作成禁止

UC#3 では数値計算が業務の中核ですが、**Pythonスクリプトを書かせず、計算過程をすべて Markdown の表と文章で書き切らせます**。これにより:

- 計算過程が誰でも読める形で残る
- 前提を変える場合も Markdown を編集するだけで意図が伝わる
- 投資判断会議で「この WACC はなぜ7%か」を議論しやすい

これは「AI を使って Excel を自動化する」のとは違う、「数値分析を言語化して組織知にする」アプローチを体験するためのルールです。詳細は手順書 §6.2 と §8.3 ③ を参照。
